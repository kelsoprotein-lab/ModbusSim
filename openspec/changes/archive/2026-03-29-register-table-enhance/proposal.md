## 为什么

当前寄存器表格存在三个阻碍日常使用的体验问题：

1. **表格无法滚动** — CSS 中直接在 `<table>` 元素上设 `overflow-y: auto` 无效（HTML 规范限制），100+ 个寄存器时无法查看完整列表，表头也无法正确 sticky
2. **无法筛选** — 只能通过左侧树按功能码类型（Coil/DI/HR/IR）过滤，没有按地址搜索或名称过滤功能。寄存器数量多时找目标地址只能肉眼扫
3. **FC3/FC4 值解析无法多选** — 当前 ValuePanel 固定从选中地址开始自动读取后续连续寄存器做 32/64 位组合，用户无法手动选择参与组合的寄存器。如果 Float32 值从 0x0004 开始但用户点了 0x0005，解析结果就是错的

## 变更内容

### 修复表格滚动
- **修复** RegisterTable.vue 中表格滚动：将 `<table>` 包裹在可滚动的 `<div>` 容器中，表头保持 sticky
- **新增** 键盘导航：支持上下箭头键在寄存器行之间移动选中状态

### 新增筛选功能
- **新增** 表格顶部搜索框，支持按地址（十六进制/十进制）和名称模糊搜索
- **修改** `filteredRegisters` 计算属性，在现有类型过滤基础上叠加搜索条件

### 新增多选与组合解析
- **新增** 表格支持 Ctrl+点击多选、Shift+点击范围选
- **修改** `selectedRegister` 从单个对象改为数组，支持传递多个选中寄存器
- **修改** ValuePanel 根据选中的寄存器数量智能解析：1 个→16 位，2 个→32 位，4 个→64 位

## 功能 (Capabilities)

### 新增功能
- `register-search`: 寄存器搜索筛选 — 表格顶部搜索框，支持按地址（十六进制/十进制）和名称模糊过滤寄存器列表
- `register-multiselect`: 寄存器多选 — 支持 Ctrl+点击逐个选/取消、Shift+点击范围选，选中的寄存器传递给 ValuePanel 进行组合解析

### 修改功能
- `register-table`: 修复滚动（包裹可滚动容器）、增加键盘上下箭头导航
- `value-panel`: 从固定读取连续寄存器改为根据用户多选的寄存器进行组合解析

## 影响

- **前端**: RegisterTable.vue（滚动修复、搜索框、多选逻辑、键盘导航）、ValuePanel.vue（多选寄存器解析逻辑重写）、App.vue（`selectedRegister` 类型从单对象变更为数组）
- **后端**: 无变更
- **依赖**: 无新增依赖
