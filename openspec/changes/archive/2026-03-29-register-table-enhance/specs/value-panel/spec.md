## 修改需求

### 需求:多选寄存器解析

ValuePanel 必须根据用户选中的寄存器数量智能切换解析模式，不再自动读取邻居寄存器。

#### 场景:选中 1 个寄存器
- **当** 用户选中 1 个 Holding Register 或 Input Register
- **那么** 面板必须显示 16 位解析（Signed、Unsigned、Hex、Binary）
- **那么** 面板禁止自动读取邻居寄存器进行 32/64 位解析

#### 场景:选中 2 个同类型寄存器
- **当** 用户选中 2 个相同类型的寄存器
- **那么** 面板必须显示 16 位解析（第一个选中寄存器）
- **那么** 面板必须显示 32 位解析（Long AB CD、Long CD AB、Float AB CD、Float CD AB），使用按地址排序的两个选中寄存器值

#### 场景:选中 4 个同类型寄存器
- **当** 用户选中 4 个相同类型的寄存器
- **那么** 面板必须显示 16 位解析（第一个选中寄存器）
- **那么** 面板必须显示 32 位解析（使用前 2 个按地址排序的寄存器）
- **那么** 面板必须显示 64 位解析（Double），使用按地址排序的 4 个寄存器值

#### 场景:选中不同类型的寄存器
- **当** 用户选中的寄存器包含不同类型（如 Holding Register 和 Input Register）
- **那么** 面板必须只显示各寄存器的 16 位解析，禁止显示 32/64 位组合解析

#### 场景:选中 Bool 类型寄存器
- **当** 用户选中 1 个或多个 Coil / Discrete Input 类型寄存器
- **那么** 面板必须显示 Bool 类型解析（true/false），禁止显示 32/64 位组合解析

#### 场景:面板标题显示
- **当** 用户选中多个寄存器
- **那么** 面板标题必须显示选中的寄存器数量和地址范围（如 "4x Holding Register @ 0x0004~0x0007"）

### 需求:无选中时的空状态

未选中任何寄存器时面板必须显示提示文本。

#### 场景:无选中
- **当** 没有选中任何寄存器
- **那么** 面板必须显示 "选择一个寄存器查看详情"
