## 新增需求

### 需求:Ctrl 多选

表格必须支持 Ctrl+点击（macOS 为 Cmd+点击）切换单行的选中状态，不影响其他已选行。

#### 场景:Ctrl 点击选中
- **当** 用户 Ctrl+点击一个未选中的行
- **那么** 该行必须被加入选中集合，其他已选行保持不变

#### 场景:Ctrl 点击取消选中
- **当** 用户 Ctrl+点击一个已选中的行
- **那么** 该行必须从选中集合中移除，其他已选行保持不变

### 需求:Shift 范围选

表格必须支持 Shift+点击进行范围选择，从上一次点击的行到当前点击的行之间的所有行全部选中。

#### 场景:Shift 点击范围选
- **当** 用户先点击第 3 行，再 Shift+点击第 7 行
- **那么** 第 3 到第 7 行必须全部被选中

### 需求:普通点击替换选中

普通点击（无修饰键）必须清空所有已选行，只选中当前点击的行。

#### 场景:普通点击
- **当** 用户已选中第 3、5、7 行，然后普通点击第 10 行
- **那么** 必须只有第 10 行被选中，其他行取消选中

### 需求:选中状态传递给 ValuePanel

选中的寄存器集合必须传递给 ValuePanel 进行解析。

#### 场景:传递多个选中寄存器
- **当** 用户选中了 2 个 Holding Register 寄存器
- **那么** ValuePanel 必须接收到包含这 2 个寄存器信息（地址、类型、值）的数组

### 需求:搜索时清空选中

搜索框内容变化时必须清空当前选中状态。

#### 场景:输入搜索关键词后选中被清空
- **当** 用户已选中若干行，然后修改搜索框内容
- **那么** 所有选中状态必须被清空
