## 新增需求

### 需求:搜索框显示

表格顶部必须包含一个搜索输入框，允许用户输入搜索关键词过滤寄存器列表。

#### 场景:搜索框展示
- **当** 用户选中了一个从站设备
- **那么** 表格顶部栏中必须显示搜索输入框，带有占位提示文本

#### 场景:搜索框为空时
- **当** 搜索框内容为空
- **那么** 必须显示全部寄存器（仅受左侧树类型过滤影响）

### 需求:按十六进制地址搜索

搜索框必须支持输入 `0x` 前缀的十六进制地址进行精确匹配。

#### 场景:输入十六进制地址
- **当** 用户在搜索框输入 `0x0042`
- **那么** 必须只显示地址为 0x0042 (66) 的寄存器

#### 场景:输入部分十六进制地址
- **当** 用户在搜索框输入 `0x00`
- **那么** 必须显示所有地址的十六进制表示中包含 `00` 的寄存器

### 需求:按十进制地址搜索

搜索框必须支持输入纯数字按十进制地址匹配。

#### 场景:输入十进制地址
- **当** 用户在搜索框输入 `42`
- **那么** 必须显示地址为 42 的寄存器以及地址十六进制表示中包含 `42` 的寄存器

### 需求:按名称搜索

搜索框必须支持按寄存器名称进行不区分大小写的模糊匹配。

#### 场景:输入名称关键词
- **当** 用户在搜索框输入 `temp`
- **那么** 必须显示名称中包含 `temp`（不区分大小写）的寄存器

### 需求:搜索与类型过滤叠加

搜索条件必须与左侧树的功能码类型过滤叠加生效。

#### 场景:先选类型再搜索
- **当** 用户在左侧树中选择 "4x Holding Registers"，然后在搜索框输入 `0x000A`
- **那么** 必须只显示 Holding Register 类型中地址为 0x000A 的寄存器
