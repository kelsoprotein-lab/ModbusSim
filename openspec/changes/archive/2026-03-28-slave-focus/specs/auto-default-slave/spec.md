## 修改需求

### 需求:创建连接时自动生成默认从站

系统在创建新的从站连接时，必须自动创建一个 slave_id=1 的默认从站设备，并预填四类寄存器定义。创建时必须支持通过 `init_mode` 参数选择初始值模式（`"zero"` 或 `"random"`），默认为 `"zero"`。

#### 场景:创建连接后自动包含从站（零值模式）
- **当** 用户创建一个端口为 5020 的从站连接，`init_mode` 为 `"zero"` 或未指定
- **那么** 该连接下必须自动存在一个 slave_id=1、名称为"从站 1"的从站设备
- **那么** 返回的连接信息中 device_count 必须为 1
- **那么** 所有寄存器值必须为 0/false

#### 场景:创建连接后自动包含从站（随机模式）
- **当** 用户创建一个端口为 5020 的从站连接，`init_mode` 为 `"random"`
- **那么** 该连接下必须自动存在一个 slave_id=1、名称为"从站 1"的从站设备
- **那么** 返回的连接信息中 device_count 必须为 1
- **那么** 寄存器值必须为随机生成

### 需求:默认从站预填寄存器定义

自动创建的默认从站必须包含 FC1/FC2/FC3/FC4 四类寄存器，每类地址范围 0~100（共 101 个地址），共 404 个 RegisterDef。此行为不受 `init_mode` 影响。

#### 场景:预填寄存器定义数量
- **当** 默认从站创建完成（无论哪种初始值模式）
- **那么** 从站必须包含 404 个 RegisterDef（Coil 101 + Discrete Input 101 + Holding Register 101 + Input Register 101）

#### 场景:预填寄存器类型和数据类型
- **当** 默认从站创建完成
- **那么** Coil 和 Discrete Input 的 RegisterDef 数据类型必须为 Bool
- **那么** Holding Register 和 Input Register 的 RegisterDef 数据类型必须为 UInt16

### 需求:前端新建连接对话框

前端新建连接对话框必须支持选择初始值模式。

#### 场景:新建连接对话框
- **当** 用户点击"新建连接"按钮
- **那么** 必须弹出对话框，包含端口号输入和初始值模式选项（全零/随机）

#### 场景:选择随机模式新建连接
- **当** 用户输入端口号并选择"随机"初始值模式后确认
- **那么** 前端必须调用 `create_slave_connection` 命令，传入 `init_mode: "random"`
