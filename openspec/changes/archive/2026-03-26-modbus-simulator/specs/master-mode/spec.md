## 新增需求

### 需求:主站TCP连接

系统必须能够以 Modbus TCP 客户端的身份连接到指定的从站设备。

#### 场景:建立连接

- **当** 用户输入目标 IP 地址 `192.168.1.100`、端口 `502`，并点击"连接"
- **那么** 系统必须建立 TCP 连接，状态显示为"已连接"

#### 场景:连接失败

- **当** 用户尝试连接但目标设备不可达
- **那么** 系统必须在超时后显示"连接失败"错误提示

#### 场景:断开连接

- **当** 用户点击"断开"
- **那么** 系统必须关闭 TCP 连接，状态显示为"已断开"

### 需求:主站读操作

系统必须支持通过功能码 01/02/03/04 读取从站的寄存器数据。

#### 场景:读取保持寄存器

- **当** 用户选择 FC=03，输入起始地址=40001、数量=10、从站 ID=1，点击"读取"
- **那么** 系统必须发送读取请求并在结果区域显示返回的 10 个寄存器值

#### 场景:读取线圈状态

- **当** 用户选择 FC=01，输入起始地址=00001、数量=8、从站 ID=1，点击"读取"
- **那么** 系统必须发送读取请求并显示 8 个线圈的 bool 值

#### 场景:读取超时

- **当** 从站在超时时间内未响应
- **那么** 系统必须显示"读取超时"错误，不阻塞后续操作

### 需求:主站写操作

系统必须支持通过功能码 05/06/15/16 向从站写入数据。

#### 场景:写单个保持寄存器

- **当** 用户选择 FC=06，输入地址=40001、值=9999、从站 ID=1，点击"写入"
- **那么** 系统必须发送写入请求并显示写入成功的确认

#### 场景:写多个保持寄存器

- **当** 用户选择 FC=16，输入起始地址=40001、值列表=[100, 200, 300]、从站 ID=1，点击"写入"
- **那么** 系统必须发送批量写入请求并显示写入成功的确认

#### 场景:写入失败

- **当** 从站返回异常响应（如非法地址）
- **那么** 系统必须显示异常码及其含义（如"Illegal Data Address"）

### 需求:主站轮询

系统必须支持对指定寄存器进行定时轮询读取。

#### 场景:启动轮询

- **当** 用户配置轮询间隔为 1000ms，选择 FC=03 读取 40001 起始的 10 个寄存器，点击"启动轮询"
- **那么** 系统必须每 1000ms 自动发送一次读取请求，并实时更新结果显示

#### 场景:停止轮询

- **当** 用户点击"停止轮询"
- **那么** 系统必须停止定时读取，保留最后一次读取的结果

#### 场景:轮询中出现通信错误

- **当** 轮询过程中某次请求超时或返回错误
- **那么** 系统必须在日志中记录错误，但继续下一次轮询，不自动停止
