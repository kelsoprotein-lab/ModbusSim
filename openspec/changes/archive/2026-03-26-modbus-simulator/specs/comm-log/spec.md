## 新增需求

### 需求:实时通信日志

系统必须实时捕获并显示所有 Modbus 通信报文，包括收发方向、时间戳、功能码和数据内容。

#### 场景:子站模式接收请求

- **当** 子站收到主站的 FC=03 读取请求（起始地址 40001，数量 2）
- **那么** 通信日志必须显示一条记录：时间戳、方向（←接收）、功能码（FC=03）、详情（R 40001 x2）

#### 场景:子站模式发送响应

- **当** 子站发送响应报文
- **那么** 通信日志必须显示一条记录：时间戳、方向（→发送）、功能码、响应数据

#### 场景:主站模式发送请求和接收响应

- **当** 主站发送读取请求并收到响应
- **那么** 通信日志必须依次显示发送记录和接收记录

### 需求:日志过滤与管理

系统必须提供日志的清除和基本管理功能。

#### 场景:清除日志

- **当** 用户点击"清除"按钮
- **那么** 系统必须清空当前显示的所有日志记录

#### 场景:日志自动滚动

- **当** 有新的日志条目产生
- **那么** 日志面板必须自动滚动到最新条目

#### 场景:日志数量上限

- **当** 日志条目超过 10000 条
- **那么** 系统必须自动移除最早的记录，保持日志条目不超过上限

### 需求:日志导出

系统必须支持将通信日志导出为文件。

#### 场景:导出日志为文本文件

- **当** 用户点击"导出"按钮
- **那么** 系统必须弹出文件保存对话框，将当前所有日志记录导出为 .txt 或 .csv 文件
