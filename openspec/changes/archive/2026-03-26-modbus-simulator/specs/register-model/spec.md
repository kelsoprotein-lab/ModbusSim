## 新增需求

### 需求:四种寄存器类型

系统必须支持 Modbus 协议定义的四种寄存器类型：Coil（线圈，0x）、Discrete Input（离散输入，1x）、Holding Register（保持寄存器，4x）、Input Register（输入寄存器，3x）。

#### 场景:创建不同类型的寄存器

- **当** 用户在从站设备中添加寄存器，选择类型为"Holding Register"，地址为 40001
- **那么** 系统必须在保持寄存器区创建该地址的寄存器条目

#### 场景:Coil寄存器只能存储布尔值

- **当** 用户创建一个 Coil 类型寄存器
- **那么** 系统必须限制其值为 true 或 false

#### 场景:子站模式下Input Register只读

- **当** 主站发送写入请求到 Input Register 地址
- **那么** 系统必须返回 Modbus 异常码 02（Illegal Data Address）

### 需求:多数据类型解读

系统必须支持将原始 16-bit 寄存器值解读为多种数据类型：bool、uint16、int16、uint32、int32、float32。跨寄存器的数据类型（32-bit）必须自动占用连续的两个寄存器地址。

#### 场景:float32类型占用两个寄存器

- **当** 用户在地址 40001 创建一个 float32 类型的寄存器，值为 25.5
- **那么** 系统必须将该值编码为 IEEE 754 格式，占用 40001 和 40002 两个寄存器地址

#### 场景:int16类型显示有符号值

- **当** 寄存器原始值为 0xFFFE，数据类型设置为 int16
- **那么** 系统必须在界面上显示值为 -2

#### 场景:uint16类型显示无符号值

- **当** 寄存器原始值为 0xFFFE，数据类型设置为 uint16
- **那么** 系统必须在界面上显示值为 65534

### 需求:字节序支持

系统必须支持四种字节序来解读跨寄存器的数据类型（uint32/int32/float32）：Big Endian (AB CD)、Little Endian (CD AB)、Mid-Big Endian (BA DC)、Mid-Little Endian (DC BA)。

#### 场景:Big Endian解读float32

- **当** 寄存器 40001=0x41C8、40002=0x0000，数据类型为 float32，字节序为 Big Endian
- **那么** 系统必须将其解读并显示为 25.0

#### 场景:切换字节序实时更新

- **当** 用户将一个 float32 寄存器的字节序从 Big Endian 切换为 Little Endian
- **那么** 系统必须立即按照新的字节序重新解读并更新显示值

### 需求:寄存器元数据

系统必须支持为每个寄存器定义名称和注释，用于界面展示和配置导出。

#### 场景:设置寄存器名称和注释

- **当** 用户为地址 40001 的寄存器设置名称为"温度"、注释为"℃"
- **那么** 系统必须在寄存器表格中显示该名称和注释

#### 场景:名称和注释可选

- **当** 用户创建寄存器但不输入名称和注释
- **那么** 系统必须正常创建寄存器，名称和注释列显示为空

### 需求:寄存器值编辑

系统必须允许用户在界面上直接编辑寄存器的值。

#### 场景:在子站模式下编辑保持寄存器值

- **当** 用户在子站模式的寄存器表格中双击 40001 的值列，输入新值 100
- **那么** 系统必须立即更新该寄存器的值，后续主站读取时返回新值

#### 场景:输入值超出数据类型范围

- **当** 用户为 uint16 类型的寄存器输入值 70000（超出 0-65535 范围）
- **那么** 系统必须提示错误，拒绝该输入
