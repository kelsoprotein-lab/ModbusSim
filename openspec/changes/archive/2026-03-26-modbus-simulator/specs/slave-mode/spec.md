## 新增需求

### 需求:子站TCP监听

系统必须能够在指定的 IP 地址和端口上启动 Modbus TCP 服务端监听，接受主站的连接请求。

#### 场景:启动子站TCP服务

- **当** 用户配置监听地址为 `0.0.0.0`、端口为 `502`，并点击"启动"
- **那么** 系统必须在该地址和端口上开始监听 TCP 连接，状态显示为"运行中"

#### 场景:停止子站TCP服务

- **当** 用户点击"停止"
- **那么** 系统必须关闭 TCP 监听，断开所有已连接的客户端，状态显示为"已停止"

#### 场景:端口被占用

- **当** 用户尝试启动监听但端口已被其他程序占用
- **那么** 系统必须显示错误提示，说明端口被占用，服务未启动

### 需求:多从站ID管理

系统必须支持在一个 TCP 连接下注册多个从站 ID（1-247），每个从站 ID 拥有独立的寄存器映射。

#### 场景:添加从站设备

- **当** 用户在一个 TCP 连接下点击"添加从站"，输入从站 ID=1 和名称"温控器"
- **那么** 系统必须创建一个新的从站设备，拥有空的寄存器映射表

#### 场景:添加重复从站ID

- **当** 用户尝试在同一连接下添加已存在的从站 ID
- **那么** 系统必须拒绝添加并提示"从站 ID 已存在"

#### 场景:删除从站设备

- **当** 用户删除一个从站设备
- **那么** 系统必须移除该从站及其所有寄存器映射，后续对该从站 ID 的请求不再响应

### 需求:子站请求响应

系统必须根据收到的 Modbus 请求中的从站 ID，将请求路由到对应的从站设备进行处理，并返回正确的响应。

#### 场景:读取已注册从站的保持寄存器

- **当** 主站发送 FC=03 请求，目标从站 ID=1，起始地址=40001，数量=2
- **那么** 系统必须从从站 ID=1 的保持寄存器映射中读取对应数据并返回正确的响应报文

#### 场景:请求未注册的从站ID

- **当** 主站发送请求的目标从站 ID 在当前连接中未注册
- **那么** 系统必须不响应该请求（静默丢弃，符合 Modbus 协议规范）

#### 场景:写入单个保持寄存器

- **当** 主站发送 FC=06 请求，目标从站 ID=1，地址=40001，值=1234
- **那么** 系统必须更新从站 ID=1 的保持寄存器 40001 的值为 1234，并返回确认响应

### 需求:多客户端连接

系统必须支持多个主站客户端同时连接到同一个子站 TCP 服务。

#### 场景:多个主站同时连接

- **当** 两个不同的主站客户端同时连接到子站服务
- **那么** 系统必须同时处理两个客户端的请求，各自独立、互不干扰

#### 场景:显示连接数

- **当** 有主站客户端连接或断开
- **那么** 系统必须在界面上实时更新当前连接的客户端数量
