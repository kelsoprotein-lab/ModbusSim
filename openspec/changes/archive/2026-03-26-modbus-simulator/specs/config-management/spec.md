## 新增需求

### 需求:配置导出

系统必须支持将当前从站设备的完整配置（连接参数、从站 ID、寄存器映射及元数据）导出为 JSON 文件。

#### 场景:导出单个从站配置

- **当** 用户选中一个从站设备，点击"导出"
- **那么** 系统必须弹出文件保存对话框，生成包含该从站所有寄存器定义、名称、数据类型、字节序、注释和当前值的 JSON 文件

#### 场景:导出整个连接配置

- **当** 用户选中一个连接，点击"导出"
- **那么** 系统必须导出该连接下所有从站设备的完整配置，包括连接参数

### 需求:配置导入

系统必须支持从 JSON 文件导入设备配置，恢复寄存器映射和元数据。

#### 场景:导入从站配置

- **当** 用户点击"导入"，选择一个有效的 JSON 配置文件
- **那么** 系统必须解析文件并创建对应的从站设备及其寄存器映射

#### 场景:导入无效文件

- **当** 用户导入一个格式错误或不符合规范的 JSON 文件
- **那么** 系统必须显示具体的错误信息，说明哪里不符合预期格式

### 需求:连接参数持久化

系统必须在退出时自动保存当前的连接配置和从站设备列表，重新启动时自动恢复。

#### 场景:关闭并重新打开应用

- **当** 用户关闭 ModbusSim 后重新打开
- **那么** 系统必须恢复上次退出时的所有连接配置和从站设备（处于停止状态）

#### 场景:首次启动

- **当** 用户首次启动 ModbusSim，无历史配置
- **那么** 系统必须显示空的初始界面，引导用户创建第一个连接
