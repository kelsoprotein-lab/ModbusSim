## 上下文

当前 ModbusSim 前端采用左侧 Sidebar + 右侧 Tab 切换的布局，从站管理的连接列表、设备列表、寄存器表格三层信息扁平堆叠在单个滚动视图中。用户创建连接后需手动添加从站设备和寄存器，才能得到可用状态。参考 Redisant MSE 的专业工具界面，需要将整体布局重构为工具栏 + 三栏 + 底部日志的经典工具型应用布局。

技术栈：Tauri v2 + Vue 3 (Composition API) + TypeScript 前端，Rust 后端。

## 目标 / 非目标

**目标：**
- 创建端口后自动生成可用的默认从站（slave_id=1, FC1/2/3/4 地址 0~100），零配置即可开始使用
- 实现 Redisant MSE 风格的三栏布局：左侧树形导航 + 中间寄存器表格 + 右侧值解析面板
- 底部常驻可折叠通信日志面板
- 顶部工具栏集中管理所有操作

**非目标：**
- Master 模式的 UI 重构（本次聚焦从站模式，Master 暂保持现状或隐藏入口）
- 串口（COM）连接支持（仅 TCP）
- 寄存器实时图表/趋势图功能
- 配置文件持久化的改动（export/import_app_state 接口不变）

## 决策

### 1. 前端布局架构

**选择**：固定三栏 + 顶部工具栏 + 底部日志面板，使用 CSS Grid 实现

**理由**：CSS Grid 可以精确控制各区域尺寸，树形导航固定宽度 240px，值面板固定宽度 280px，中间表格自适应。底部日志面板支持拖拽调整高度。这比 flexbox 嵌套更直观，也更接近 Redisant MSE 的布局行为。

**替代方案**：使用 splitpanes 库实现可拖拽分割面板 — 功能更强但引入额外依赖，初期不需要。

### 2. 树形导航的实现

**选择**：自定义 Vue 组件实现，不引入第三方树组件库

**理由**：树的层级固定为三层（连接 > 从站 > 寄存器分组），结构简单且确定，不需要通用树组件的复杂功能（拖拽排序、懒加载等）。自定义实现可以精确控制每个节点的右键菜单和图标样式。

**替代方案**：使用 vue-treeselect 或 el-tree — 过于通用，样式定制成本高。

### 3. 寄存器数据的内存模型

**选择**：后端 RegisterMap (HashMap) 保持不变，前端通过现有 `list_registers` + `read_register` 命令获取数据

**理由**：当前 RegisterMap 使用 HashMap 存储，读取不存在的地址返回默认值 0/false，天然支持稀疏寄存器。自动创建时只需在后端循环添加 RegisterDef 定义，不需要改变存储结构。

**替代方案**：将 RegisterMap 改为连续数组预分配 — 内存效率更高但需要重构核心数据结构，风险大。

### 4. 值解析面板的计算方式

**选择**：在前端 TypeScript 中实现值的多格式解析，不走后端 IPC

**理由**：值解析是纯计算逻辑（位运算、IEEE754 转换），无需后端参与。每次选中寄存器行时，前端读取当前值和相邻寄存器值（用于 32/64 位组合），在本地计算所有格式。避免频繁 IPC 调用造成的延迟。

**替代方案**：在后端 Rust 中实现解析命令 — 类型安全更好但增加 IPC 开销，对于纯展示逻辑不值得。

### 5. 自动创建默认从站的时机

**选择**：在后端 `create_slave_connection` 命令中同步完成

**理由**：作为原子操作，保证创建连接后立即返回包含默认从站的完整状态。前端无需在创建连接后再发起额外请求。如果放在前端串行调用，网络延迟和错误处理会变复杂。

### 6. 底部日志面板

**选择**：集成到主布局的 CSS Grid 中，使用拖拽手柄调整高度，默认折叠

**理由**：Redisant MSE 的日志面板是底部常驻的，随时可查看通信数据。默认折叠可以给寄存器表格更多空间，用户需要时展开。

## 风险 / 权衡

- **[性能]** 自动创建 404 个寄存器定义（4类 x 101 地址）可能在初始化时有短暂延迟 → 缓解：101 个地址的 HashMap 插入在 Rust 中是微秒级，不会有感知
- **[兼容性]** 布局重构是 BREAKING 变更，现有的 Master 模式和 Tools 页面需要新的入口 → 缓解：工具栏提供 Tools 下拉菜单；Master 模式暂通过工具栏按钮或菜单访问
- **[前端复杂度]** 三栏布局 + 树组件 + 值面板是显著的前端工作量 → 缓解：分步实现，先完成布局骨架和树组件，再填充功能
- **[数据量]** 展示 101 行寄存器表格时 DOM 节点较多 → 缓解：101 行在现代浏览器中无性能问题；如未来扩展到更大范围可加虚拟滚动
