## 新增需求

### 需求:选中寄存器时显示多格式值解析

用户在寄存器表格中选中一行时，右侧值解析面板必须显示该寄存器地址的值的多种数据格式解读。面板宽度固定为 280px。

#### 场景:显示16位值解析
- **当** 用户选中一个 Holding Register 地址 0x0000，值为 881
- **那么** 值解析面板必须显示以下格式：
  - Signed: 881
  - Unsigned: 881
  - Hex: 0x0371
  - Binary: 0000 0011 0111 0001

#### 场景:显示32位组合值解析
- **当** 用户选中一个 Holding Register 地址 0x0000，且相邻地址 0x0001 也存在值
- **那么** 值解析面板必须组合两个连续寄存器显示32位解析：
  - Long AB CD: 两个寄存器按 Big Endian 组合的十进制值
  - Long CD AB: 两个寄存器按 Little Endian 组合的十进制值
  - Float AB CD: Big Endian 的 IEEE754 浮点值
  - Float CD AB: Little Endian 的 IEEE754 浮点值

#### 场景:显示64位组合值解析
- **当** 用户选中的寄存器地址后有连续 3 个相邻地址存在值
- **那么** 值解析面板必须组合四个连续寄存器显示64位解析：
  - Double: IEEE754 双精度浮点值

### 需求:Bool类型寄存器的值解析

当选中 Coil 或 Discrete Input 类型寄存器时，值解析面板必须适配布尔值的展示。

#### 场景:显示Bool值
- **当** 用户选中一个 Coil 类型寄存器，值为 true
- **那么** 值解析面板必须显示：
  - Value: true (1)
  - Hex: 0x0001

### 需求:未选中时的空状态

当寄存器表格中没有选中任何行时，值解析面板必须显示空状态提示。

#### 场景:无选中行
- **当** 没有选中任何寄存器行
- **那么** 值解析面板必须显示提示文本 "选择一个寄存器查看详情"

### 需求:值解析面板标题

值解析面板顶部必须显示当前选中寄存器的类型和地址信息。

#### 场景:显示标题信息
- **当** 用户选中 4x Holding Register 地址 0x0000
- **那么** 面板标题必须显示 "4x Holding Register @ 0x0000"
