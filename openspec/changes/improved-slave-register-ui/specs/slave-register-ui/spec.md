## 新增需求

### 需求:添加寄存器对话框

当用户点击"添加寄存器"按钮时，系统必须显示一个模态对话框，让用户填写寄存器信息。对话框必须包含：名称（文本输入）、地址（数字输入）、类型（下拉选择：保持寄存器/输入寄存器/线圈/离散输入）。高级选项（数据类型、字节序、注释）必须默认折叠。

### 需求:寄存器右键菜单

当用户在寄存器表格中的任意行上右键点击时，系统必须显示上下文菜单，菜单必须包含以下选项：
- 编辑寄存器
- 删除寄存器

#### 场景:右键点击显示菜单

- **当** 用户在寄存器表格的行上右键点击
- **那么** 系统必须在鼠标指针位置显示上下文菜单，菜单项为"编辑寄存器"和"删除寄存器"

#### 场景:右键菜单外点击

- **当** 上下文菜单显示后，用户在菜单区域外点击
- **那么** 系统必须关闭上下文菜单

### 需求:编辑寄存器

当用户选择"编辑寄存器"时，系统必须复用水添加寄存器对话框，并预填该寄存器的当前所有属性值（名称、地址、类型、数据类型、字节序、注释）。用户确认修改后，系统必须更新后端寄存器定义。

#### 场景:编辑后确认保存

- **当** 用户在编辑寄存器对话框中修改了寄存器信息并点击"保存"
- **那么** 系统必须更新该寄存器定义，后端返回成功响应

### 需求:删除寄存器

当用户选择"删除寄存器"时，系统必须直接从后端移除该寄存器定义，无需弹出确认对话框。

#### 场景:删除后列表更新

- **当** 用户在右键菜单中点击"删除寄存器"
- **那么** 系统必须立即从寄存器列表中移除该行，并调用后端删除接口

### 需求:地址冲突检测

当用户在添加或编辑寄存器时，输入的地址在该从站同类型寄存器中已存在，系统必须弹出提示："该地址已存在，是否覆盖？"，提供"覆盖"和"取消"两个选项。

#### 场景:检测到地址冲突

- **当** 用户填写了与现有寄存器相同的地址和类型，并点击"确认"
- **那么** 系统必须弹出冲突提示，显示"该地址已存在，是否覆盖？"以及"覆盖"/"取消"按钮

#### 场景:用户选择覆盖

- **当** 系统显示地址冲突提示后，用户点击"覆盖"
- **那么** 系统必须用新值覆盖原有寄存器定义，关闭对话框

#### 场景:用户选择取消

- **当** 系统显示地址冲突提示后，用户点击"取消"
- **那么** 系统必须关闭冲突提示，用户停留在对话框中继续修改

### 需求:高级选项默认折叠

数据类型（UInt16/Int16/UInt32/Int32/Float32/Bool）和字节序（Big/Little/MidBig/MidLittle）选项在添加/编辑寄存器对话框中必须默认隐藏，用户点击"显示高级选项"后才会显示。

#### 场景:默认隐藏高级选项

- **当** 用户打开添加寄存器对话框
- **那么** 表单中必须只显示名称、地址、类型三个字段，高级选项区域必须被折叠

#### 场景:展开高级选项

- **当** 用户点击"显示高级选项"
- **那么** 系统必须展开高级选项区域，显示数据类型、字节序、注释三个字段

### 需求:UI全中文

界面上所有文本标签、按钮、菜单项必须使用中文。按钮文案示例：
- 添加寄存器 / 编辑寄存器 / 删除寄存器 / 保存 / 取消 / 覆盖
- 显示高级选项 / 隐藏高级选项
- 寄存器类型下拉项：保持寄存器、输入寄存器、线圈、离散输入
- 数据类型下拉项：Bool、UInt16、Int16、UInt32、Int32、Float32
- 字节序下拉项：大端序（Big）、小端序（Little）、中大端序（Mid-Big）、中小端序（Mid-Little）
