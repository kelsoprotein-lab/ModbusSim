## 新增需求

### 需求:Modbus地址转换

系统必须提供 Modbus 协议地址与 PLC 地址之间的双向转换工具。

#### 场景:协议地址转PLC地址

- **当** 用户输入 Modbus 协议地址 0x0000（Holding Register）
- **那么** 系统必须显示对应的 PLC 地址 40001

#### 场景:PLC地址转协议地址

- **当** 用户输入 PLC 地址 40001
- **那么** 系统必须显示对应的 Modbus 协议地址 0x0000（Holding Register）和寄存器类型

#### 场景:输入无效地址

- **当** 用户输入不合法的地址格式
- **那么** 系统必须显示错误提示

### 需求:CRC/LRC校验计算

系统必须提供 Modbus RTU CRC-16 和 Modbus ASCII LRC 校验值的计算工具。

#### 场景:计算CRC校验

- **当** 用户输入十六进制数据 `01 03 00 00 00 0A`
- **那么** 系统必须计算并显示 CRC-16 校验值

#### 场景:计算LRC校验

- **当** 用户输入十六进制数据 `01 03 00 00 00 0A`
- **那么** 系统必须计算并显示 LRC 校验值

#### 场景:输入格式灵活

- **当** 用户输入数据时使用空格、逗号或无分隔符（如 `010300000001`）
- **那么** 系统必须能够正确解析并计算校验值
