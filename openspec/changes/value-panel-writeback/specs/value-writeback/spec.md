## 新增需求

### 需求:双击解析值进入编辑模式
用户必须能够通过双击 ValuePanel 中的解析值文本进入编辑模式。编辑模式下，原始文本必须替换为输入框，输入框内必须预填当前解析值。

#### 场景:双击 16-bit Signed 值
- **当** 用户双击 ValuePanel 中 Signed 行的值文本
- **那么** 该值文本替换为输入框，预填当前 Signed 解析值

#### 场景:双击 32-bit Float AB CD 值
- **当** 用户选中了 2 个同类型寄存器，双击 Float AB CD 行的值文本
- **那么** 该值文本替换为输入框，预填当前 Float AB CD 解析值

#### 场景:寄存器数量不足时不可编辑
- **当** 用户只选中了 1 个寄存器，32-bit 区域的值显示为 "-"
- **那么** 双击该行禁止进入编辑模式

### 需求:编辑确认与取消
编辑模式下，用户必须能够通过 Enter 键或失焦（blur）确认编辑，通过 Escape 键取消编辑。同一时间禁止有多个字段处于编辑状态。

#### 场景:Enter 确认
- **当** 用户在编辑输入框中按下 Enter 键
- **那么** 系统执行反向解析并写入寄存器，退出编辑模式

#### 场景:Escape 取消
- **当** 用户在编辑输入框中按下 Escape 键
- **那么** 系统丢弃编辑内容，恢复为只读显示，寄存器值不变

#### 场景:失焦确认
- **当** 输入框失去焦点
- **那么** 系统执行反向解析并写入寄存器，退出编辑模式

#### 场景:选区变化取消编辑
- **当** 用户正在编辑某个解析值，此时选中的寄存器发生变化
- **那么** 系统必须取消当前编辑，不写入任何值

### 需求:16-bit 反向解析与写入
系统必须支持将 16-bit 解析值反向转换为单个寄存器原始值并写入。

#### 场景:Signed 负值写入
- **当** 用户输入 Signed 值 -100
- **那么** 系统必须将其转换为无符号值 65436 (0xFF9C)，写入 sortedRegs[0] 对应的寄存器

#### 场景:Unsigned 值写入
- **当** 用户输入 Unsigned 值 1000
- **那么** 系统必须将 1000 写入 sortedRegs[0] 对应的寄存器

#### 场景:Hex 值写入
- **当** 用户输入 Hex 值 "0x03E8"
- **那么** 系统必须将其解析为 1000，写入 sortedRegs[0] 对应的寄存器

### 需求:32-bit 反向解析与写入
系统必须支持将 32-bit 解析值反向转换为两个 16-bit 寄存器值并写入。

#### 场景:Long AB CD 写入
- **当** 用户输入 Long AB CD 值 100000
- **那么** 系统必须将高 16 位写入 sortedRegs[0]，低 16 位写入 sortedRegs[1]

#### 场景:Long CD AB 写入
- **当** 用户输入 Long CD AB 值 100000
- **那么** 系统必须将低 16 位写入 sortedRegs[0]，高 16 位写入 sortedRegs[1]

#### 场景:Float AB CD 写入
- **当** 用户输入 Float AB CD 值 3.14
- **那么** 系统必须按 IEEE 754 编码为 4 字节，高字写入 sortedRegs[0]，低字写入 sortedRegs[1]

#### 场景:Float CD AB 写入
- **当** 用户输入 Float CD AB 值 3.14
- **那么** 系统必须按 IEEE 754 编码为 4 字节，低字写入 sortedRegs[0]，高字写入 sortedRegs[1]

### 需求:64-bit 反向解析与写入
系统必须支持将 64-bit Double 值反向转换为四个 16-bit 寄存器值并写入。

#### 场景:Double 写入
- **当** 用户选中了 4 个同类型寄存器，输入 Double 值 3.141592653589793
- **那么** 系统必须按 IEEE 754 双精度编码为 8 字节，依次写入 sortedRegs[0..3]

### 需求:Bool 值 toggle
用户必须能够通过双击 Bool 类型寄存器的值进行 toggle（0↔1），无需输入框。

#### 场景:Bool toggle
- **当** 用户双击 Bool 类型寄存器当前显示 "true (1)" 的值
- **那么** 系统必须将值写为 0，显示更新为 "false (0)"

### 需求:无效输入处理
系统必须处理无效的用户输入，禁止将无效值写入寄存器。

#### 场景:非数字输入
- **当** 用户在 Signed 字段输入 "abc" 后按 Enter
- **那么** 系统必须忽略该输入，退出编辑模式，寄存器值不变
