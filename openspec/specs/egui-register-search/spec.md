### 需求:寄存器视图 always-visible 搜索框

`Selection::RegisterGroup` 视图的 heading 行右端必须固定一个 `TextEdit` 搜索框，宽度约 160 px，始终可见（不得需要点击展开）。该搜索框的 placeholder 必须为"地址 / 名称…"。搜索框必须只在"选中了具体的 RegisterGroup"时出现；其他 Selection 分支（None / Connection / Device）不得显示。

搜索框的状态（当前输入文本）必须保存在 `SlaveApp` 级，key 为 `(conn_id, slave_id, reg_type)` 三元组；切换 RegisterGroup 时不得继承上一次的文本。

#### 场景:FC03 视图默认显示搜索框

- **当** 用户选中任一 Device 下的 FC03 保持寄存器组
- **那么** 在 heading 行"批量添加"按钮左侧必须可见一个空的 `TextEdit`
- **那么** 其 placeholder 为"地址 / 名称…"

#### 场景:切换 RegisterGroup 清空搜索

- **当** 用户在 FC03 搜索了"1234"然后点击 FC04
- **那么** FC04 视图的搜索框必须为空
- **那么** 切回 FC03 后必须保留"1234"文本（按 `(conn_id, slave_id, reg_type)` key 持久）

### 需求:地址跳转（纯数字输入）

当搜索框输入为合法 `u16`（十进制 0..=65535 或以 `0x`/`0X` 开头的十六进制）时，系统必须：
1. 将输入解析为目标地址 `addr`
2. 若 `addr` 在当前 RegisterGroup 地址范围内，调用 `ScrollArea::scroll_to_rect` 将该行滚动至视口中部
3. 触发该行的"高亮"效果：`bg_selected` 填色持续 2 秒后淡出
4. 不得对表格应用过滤（即其它行仍可见）

地址解析必须不区分大小写（`0xFF` / `0xff` 均合法）。非法输入（非数字字符、超出 u16 范围）必须不触发跳转，并且**不得**显示错误气泡（由过滤路径兜底，见下条需求）。

#### 场景:十进制地址跳转

- **当** 用户在 FC03 搜索框输入"1234"并按回车（或输入后停顿 200ms）
- **那么** 表格必须自动滚动到 addr=1234 这一行
- **那么** 该行必须在 2 秒内有蓝色高亮 fade
- **那么** 其他行依然可见（非过滤）

#### 场景:十六进制地址跳转

- **当** 用户输入"0xABCD"
- **那么** 必须解析为 addr=43981 并滚动到该行

#### 场景:超出范围的数字回退过滤

- **当** 用户输入"99999"（超 u16）
- **那么** 系统必须不做地址跳转
- **那么** 走"名称/注释过滤"路径（99999 作为字符串匹配）
- **那么** 无匹配则表格显示为空列表 + "无结果"提示

### 需求:名称 / 注释模糊过滤

当搜索框输入**非纯数字**或**数字超出 u16 范围**时，系统必须对表格应用**大小写不敏感**的子字符串过滤：对每行的 `name` 与 `comment` 字段做 `contains()` 检查，命中则显示，否则隐藏。

过滤结果为空时表格必须显示占位文字"无匹配寄存器"，而非空白。

#### 场景:按名称过滤

- **当** 用户输入"Temp"且某些寄存器 name 为"TempSensor1"、"TempSensor2"
- **那么** 表格必须只显示 name 含"temp"（不区分大小写）的行
- **那么** 行号列必须仍显示原始地址（不重新编号）

#### 场景:空结果提示

- **当** 用户输入"XYZ"且无任何寄存器匹配
- **那么** 表格区必须显示灰色提示"无匹配寄存器"
- **那么** 不得显示错误弹窗

### 需求:Cmd+F / Ctrl+F 快捷键聚焦搜索框

在 `Selection::RegisterGroup` 视图激活的前提下，按下 macOS `Cmd+F` 或其他平台 `Ctrl+F` 必须将键盘焦点移至该视图的搜索框，并**全选**已有文本便于覆盖输入。

该快捷键必须在 egui 侧 consume（通过 `ctx.input_mut(|i| i.consume_shortcut(...))`），防止被宿主窗口系统拦截（macOS 下默认可能触发"关闭窗口"）。

当用户在非 `Selection::RegisterGroup` 视图按下该快捷键时，系统必须不做任何处理（不得聚焦到其他地方）。

#### 场景:macOS Cmd+F 聚焦

- **当** 用户在 FC03 视图按下 Cmd+F
- **那么** 搜索框必须获得键盘焦点
- **那么** 已有内容（若有）必须被全选
- **那么** 宿主窗口系统不得收到该按键事件

#### 场景:非 RegisterGroup 视图不响应

- **当** 用户当前 Selection 为 Device 或 None 且按下 Cmd+F
- **那么** 系统必须不做响应
- **那么** egui 必须不 consume 该事件（留给其他系统 default handler）
